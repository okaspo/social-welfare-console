# Chat-First UX方針

## 核心コンセプト: 「AIが主役」

ユーザーがフォームに入力するのではなく、**AIチャットを通じて資料を作成する**体験を基本とする。

## 設計原則

### 1. チャットが起点
- ユーザーは「理事会の議事録を作成して」と話しかける
- AIが必要な情報を対話で聞き取る
- AIがCanvasに内容を入力していく（ユーザーは見守る）

### 2. AI主導の入力
- 従来: ユーザーがフォームに直接入力
- 新方式: AIがユーザーとの対話から情報を抽出し、自動入力
- Canvasはリアルタイムで更新される（AIがタイピングしているように見える）

### 3. 手動編集も可能
- AIが入力した内容はユーザーが直接編集可能
- 「この部分を修正して」とチャットで依頼することもできる
- 両方のモードをシームレスに行き来できる

## 実装パターン

```
【対話例】

ユーザー: 「議事録を作成して」
葵: 「かしこまりました。いつの会議の議事録でしょうか？」
ユーザー: 「今日の理事会」
葵: 「本日(12/21)の理事会ですね。出席者を教えていただけますか？」
ユーザー: 「山田理事長、佐藤理事、鈴木監事が出席」
葵: 「ありがとうございます。議題は何でしたか？」

（この間、Canvasには情報がリアルタイムで入力されていく）

葵: 「議事録の下書きができました。内容を確認してください。
     修正があれば『○○を△△に変更して』とお伝えください」
```

## 技術的実装

1. **ストリーミング更新**: AIの回答と同時にCanvasを更新
2. **構造化データ抽出**: 対話から日付、人名、議題を抽出
3. **Toolsによる操作**: `update_canvas_field` ツールでフィールド更新
4. **履歴管理**: 変更履歴を保持し、Undo/Redo可能に

## 禁止事項

- ユーザーに長いフォームを最初から埋めさせない
- チャットを補助的な存在にしない（チャットが主役）
- AI入力中にユーザーの操作をブロックしない

---

> **この方針は全ての新しいUI/UX設計に適用すること**
